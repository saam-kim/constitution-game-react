# 헌법 제정 게임

고등학교 2학년 사회 수업에서 롤스의 **무지의 베일** 사고실험을 체험하기 위한 React + Firebase 기반 시뮬레이션 게임입니다.

학생들은 자신이 어떤 계층이나 처지로 태어날지 모르는 상태에서 헌법의 핵심 정책 방향을 정합니다. 이후 룰렛으로 “미래의 나”가 공개되면, 처음 선택한 규칙이 가장 불리한 위치의 시민에게도 받아들일 만한지 검토하고 헌법을 수정합니다.

## 배포 주소

[https://saam-kim.github.io/constitution-game-react/](https://saam-kim.github.io/constitution-game-react/)

GitHub Pages에서는 새로고침 404 오류를 피하기 위해 해시 라우팅을 사용합니다.

## 수업 흐름

1. **토론**: 학생 입장, 모둠 확정, 쟁점 확인
2. **헌법 제정**: 모둠별 정책 방향 선택과 제출
3. **결과 확인**: 미래의 나 룰렛 공개, 1차 결과 확인
4. **새 헌법 토론**: 공개된 위치를 바탕으로 정책 재검토
5. **최종 결과 및 발표**: 같은 미래의 나 기준으로 수정 헌법 결과 비교와 발표

## 정책 선택지

학생 화면에서는 숫자를 직접 입력하지 않고, 각 정책의 방향을 선택합니다.

### 세금 정책 방향

- **낮은 세금형**: 개인과 기업의 경제 활동 자유를 우선합니다.
- **공동 부담형**: 모두가 감당 가능한 수준으로 넓게 부담합니다.
- **능력 부담형**: 소득이 높은 시민이 더 큰 책임을 집니다.

### 국가 예산 방향

- **성장 우선형**: 투자, 기업 활동, 일자리 확대를 우선합니다.
- **기본 보장형**: 생계, 의료, 주거 등 최소 생활 보장을 우선합니다.
- **기회 투자형**: 교육, 직업훈련, 돌봄 등 다시 올라설 기회에 투자합니다.

### 최저임금 방향

- **시장 자율형**: 임금을 시장과 고용 상황에 더 맡깁니다.
- **점진 인상형**: 노동자 보호와 사업자 부담을 함께 고려합니다.
- **생활 보장형**: 일하는 시민의 기본 생활 보장을 우선합니다.

## 교사용 운영 기능

- 학생 입장 후 `모둠 확정` 버튼으로 실제 참여 모둠만 남길 수 있습니다.
- 모둠 확정 전에는 타이머가 움직이지 않고, 교사가 `수업 시작`을 눌러야 시작됩니다.
- 단계 버튼으로 `토론 → 헌법 제정 → 결과 확인 → 새 헌법 토론 → 최종 결과 및 발표` 흐름을 제어합니다.
- 결과 확인 단계에서 모둠별 결과를 전환해 볼 수 있습니다.
- 1차 룰렛은 미래의 나를 공개하는 장치입니다.
- 최종 단계에서는 2차 룰렛을 돌리지 않고, 같은 미래의 나 기준으로 수정 헌법 결과를 확인합니다.
- `CSV 저장`으로 모둠별 선택, 미래의 나, 결과 점수, 뉴스/사건 카드를 저장할 수 있습니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 Vite가 표시하는 주소로 접속합니다.

## Firebase 설정

1. `.env.example` 파일을 복사해서 `.env` 파일을 만듭니다.
2. Firebase 콘솔에서 Web App 설정값을 확인합니다.
3. `.env`에 `VITE_FIREBASE_DATABASE_URL`을 포함한 값을 입력합니다.
4. Firebase Realtime Database를 활성화합니다.

Firebase 설정 전에는 로컬 데모 모드로 동작합니다. 이 모드에서는 같은 브라우저의 `localStorage`에 세션이 저장되므로 개발 중 흐름 리허설에 적합합니다. 실제 여러 기기 접속 수업은 Firebase 설정 후 사용합니다.

GitHub Pages 배포에서는 GitHub Actions Variables에 `VITE_FIREBASE_*` 값을 등록해 Firebase 설정을 주입합니다.
